# Архитектура Rodex

## Что это за проект

Rodex - веб-приложение для учёта персон, родственных связей и архивных источников. Основная функция - построение родословной росписи по системе Соса–Страдоница.

## Общая схема

В проекте используется следующая схема: React-фронтенд общается с FastAPI-бэкендом по HTTP/JSON, бэкенд работает с PostgreSQL через SQLAlchemy.

```text
React + TypeScript (frontend)
         |
         | HTTP / JSON (REST)
         v
FastAPI (backend)
         |
         | SQLAlchemy / Alembic
         v
PostgreSQL
```

Запуск настроен через Docker Compose.

## Модель данных

### User

Учётная запись. Поля: email, хэш пароля, полное имя, флаг активности. Все остальные записи привязаны к `owner_id` этого пользователя.

### Person

Персона родословной. Поля: фамилия, имя, отчество, варианты имени, пол, дата и место рождения, дата и место смерти, примечания.

### Relationship

Связь родитель–ребёнок. Поля: `parent_id`, `child_id`, `parent_role` (father / mother / unknown). При создании проверяются циклы и дублирование роли - у ребёнка не может быть двух отцов или двух матерей.

### Source

Архивный источник. Поля: заголовок, архивный шифр, URL, комментарий к достоверности, примечания.

### PersonSource

Связующая таблица между `Person` и `Source`. Пара `(source_id, person_id)` уникальна.

### RelationshipSource

То же самое, но для `Relationship`. Уникальная пара `(source_id, relationship_id)`.

## Изоляция данных

Все защищённые эндпоинты требуют JWT-токен. Зависимость `get_current_user` достаёт пользователя из токена, и все запросы к базе фильтруются по `owner_id == current_user.id`. Если обратиться к чужой записи - вернётся `404`.

## Алгоритм Соса–Страдоница

1. Пробанд получает номер `1`.

2. Для персоны с номером `n`:
   - отец получает номер `2n`;
   - мать получает номер `2n + 1`.

3. Если сведения о родителе отсутствуют, соответствующая ветвь не строится.

4. При обнаружении циклической связи проблемная ветвь пропускается, а в результат добавляется предупреждение.

5. Результат формируется по поколениям и отображается:
   - в виде списка;
   - в виде визуального дерева предков.

## Экспорт CSV

Роспись можно скачать через `GET /api/v1/reports/sosa/{proband_id}/export.csv`. В файле - строка на каждую персону: номер Соса, ФИО, пол, даты и места рождения и смерти.

## PWA

В проекте есть `manifest.webmanifest` и service worker. Service worker кэширует статические ресурсы - приложение частично работает без сети и его можно добавить на домашний экран телефона.
